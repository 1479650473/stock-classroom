## 一、选股全流程（七步）

```
① 市场环境评估 → ② 硬性过滤 → ③ 初选池 → ④ 技术面深化 → ⑤ 资金面验证 → ⑥ 情绪面估算 → ⑦ 风控排序
```

---

## 二、每一步需要的数据

### ① 市场环境评估

**目的：** 判断今天该不该买、能买多少

| 数据 | 来源 | 字段 | 当前状态 |
|------|------|------|:--:|
| 上证指数涨跌 | kline.db → `market_daily` | 无此表（只有881行 `lu_count`） | ❌ 缺失 |
| 涨跌家数 | AKShare `stock_zh_a_spot_em()` 或缓存 | 全市场涨幅分布 | ❌ 需实时行情 |
| 涨停/跌停家数 | AKShare 涨停板接口 | 当前不可用 | ❌ |
| 成交量变化 | 上证指数日K线 | `kline_daily` → market K线 | ❌ 无market表 |

**当前可用方案：** 用 `market_daily` 的 `lu_count` 列（涨停数？），或直接用简易替代——看 `kline_daily` 里最近一天所有股票的平均涨跌幅分布。或者**先跳过这一步，等缓存层填充后再接**，默认按"结构性市场"（仓位30-50%）。

**需要新增数据：** 指数日K线（上证/深成/创业板）存入 `market_daily`

---

### ② 硬性过滤（六条红线）

| 过滤条件 | 需要的数据 | 来源 | 字段 |
|------|------|------|------|
| ST / *ST | 股票名称 | `stock_list` → `name` | ✅ 已有 |
| 停牌 | 最近K线日期 | `stock_list` → `last_date` | ✅ 已有 |
| 日成交额 < 5000万 | 最近一天K线成交量 | `kline_daily` → `volume × close` | ✅ 已有 |
| 股价 < 2元 | 最近一天收盘价 | `kline_daily` → `close` | ✅ 已有 |
| 上市不足60日 | 上市日期 | `stock_list` → `listed_date` | ✅ 已有 |
| 连续跌停 | 最近3天涨跌幅 | `kline_daily` → 连续跌停检测 | ✅ 可算 |

**结论：** 硬性过滤 100% 可用本地数据，无需新增。

---

### ③ 初选池

**目的：** 从 5000+ 只缩小到几百只候选

| 筛选条件 | 需要的数据 | 字段 | 来源 |
|------|------|------|------|
| 今日涨幅 > 3%（不追涨停） | 涨跌幅 | `close` 对比 `prev_close` | `kline_daily` ✅ |
| 量比 > 1.2 | 当日量 / 5日均量 | `volume` | `kline_daily` ✅ |
| 价格 > MA10 | MA10 | `indicators.py` → `calc_ma(10)` | 可算 ✅ |
| 成交额 > 5000万 | `volume × close` | `kline_daily` | ✅ |

**结论：** 初选池 100% 可用本地数据。

---

### ④ 技术面深化（40% 权重）

**子维度：** 动量(30%) + 趋势(30%) + 形态(20%) + 金叉(20%)

#### 动量 (30%)
| 指标 | 数据 | 算法 |
|------|------|------|
| 5日涨幅 | `close[-5:]` | (close[-1]-close[-5])/close[-5] |
| 10日涨幅 | `close[-10:]` | 同上 |
| 20日涨幅 | `close[-20:]` | 同上 |
| 连续上涨天数 | `close` 逐日对比 | 统计连阳 |

✅ **全部可用 kline_daily → close**

#### 趋势 (30%)
| 指标 | 数据 | 算法 |
|------|------|------|
| 均线多头排列 | MA5/10/20/60 | `calc_ma()` 对比 |
| 价格站上均线 | close vs MA5/MA20 | 同上 |
| MA60 斜率 | MA60[-1] vs MA60[-5] | 5日变化率 |
| 新高情况 | 20日/60日高点 | `high` max |

✅ **全部可算（需要 60+ 天 K线）**

#### 形态 (20%)
| 指标 | 数据 | 算法 |
|------|------|------|
| 长下影线（锤子线）| open/high/low/close | 下影线比例 |
| 突破20日新高 | high[-20:] | close >= high[-20:] * 0.98 |
| 支撑位反弹 | MA20 + low/close | low 触及 MA20 后收阳 |
| 大阳线 | open/close | body/(high-low) > 0.7 |

✅ **全部可用 kline_daily → open/high/low/close**

#### 金叉 (20%)
| 指标 | 数据 | 算法 |
|------|------|------|
| MACD 金叉 | `indicators.py` → dif/dea/macd | dif 上穿 dea |
| RSI 合理区间 | `indicators.py` → rsi14 | 40-60 区间 |
| MA金叉 | MA5/MA10 | MA5 上穿 MA10 |
| KDJ（可选） | 需自算 | close/low/high 9日 |

✅ **全部可算（indicators.py 已覆盖 MACD/RSI/MA）**

---

### ⑤ 资金面验证（20% 权重）

| 指标 | 数据 | 算法 |
|------|------|------|
| 量比（当日/20日均量）| volume[-20:] | volume[-1]/mean(last20) |
| 量价配合 | close + volume | 涨+放量 = 健康 |
| 连续放量 | volume 最近3天 | mean(last3) > mean(last20)*1.3 |
| 换手率估算 | amount + close | amount/(close*1亿) ≈ 估算换手率 |

✅ **全部可算**

**缺失：**
| 指标 | 需要 | 当前 |
|------|------|:--:|
| 大单净流入 | AKShare `stock_individual_fund_flow_rank` | ❌ 东财反爬，待找替代 |
| 北向资金流向 | `north_flow` 表 | ✅ 2026-07-08 已入库 2704行 |

---

### ⑥ 情绪面估算（30% 权重）

| 指标 | 数据 | 算法 |
|------|------|------|
| 量比热度 | volume | 量比 > 3 → 关注度高 |
| 换手率活跃度 | 同上估算 | 5-20% → 活跃 |
| 涨幅排名 | 全市场排序 | 在所有股票中排位 |
| 涨跌停情况 | 当日涨幅分布 | 极端涨幅检测 |
| 连阳天数 | close 逐日 | 连续收阳 > 3天 |

✅ **大部分可算（换手率需要估算）**

**缺失：**
| 指标 | 需要 | 当前 |
|------|------|:--:|
| 新闻舆情 | AKShare `stock_news_em` | ✅ 已接入（个股级，按需拉） |
| 板块热度 | `sector_data` 表 | ⚠️ 接口OK但表空（交易时段才拉） |
| 龙虎榜/游资 | `lhb_stock` 表 | ✅ 2026-07-08 已入库 1038行 |
| 涨停板 | `limit_up` 表 | ✅ 2026-07-08 已入库 47行 |
| 股东人数变化 | AKShare `stock_zh_a_gdhs` | ✅ 已接入（个股级，按需拉） |
| 人气榜 | AKShare `stock_hot_rank_em` | ❌ 东财反爬，待找替代 |

## 四、按优先级的数据补全计划

### 必须补（Phase 1 — 让选股引擎能跑完整）

1. **指数日K线** — 上证/深成/创业板的历史K线和最新数据存入 `kline.db` → 支撑市场环境评估
2. **全市场实时行情缓存** — `stock_spot` 表从 20行 → 5000+行 → 支撑涨幅分布、排名等

### 应该补（Phase 2 — 提升评分质量）

3. **行业板块数据** — AKShare `stock_board_industry_name_em()` → `sector_data` → 板块联动分析
4. **北向资金** — 重新填充 `north_flow` → 资金面辅助信号
5. **个股资金流向** — AKShare → `fund_flow` → 主力动向

### 锦上添花（Phase 3）

6. 涨停板/龙虎榜 → 情绪面极端信号
7. 财报数据 (PE/PB/ROE) → 基本面评分

## 五、当前可用的简化版评分

**不依赖任何缺失数据，纯 kline.db + indicators.py：**

```
综合得分 = 技术面(50%) + 资金面(25%) + 情绪面(20%) + 基本面(5%)

技术面 = 动量(30%) + 趋势(30%) + 形态(20%) + 金叉(20%)
  全用 kline_daily + indicators.py 计算

资金面 = 量比(40%) + 量价配合(40%) + 连续放量(20%)
  全用 kline_daily volume 计算

情绪面 = 量比热度(40%) + 涨幅排名(30%) + 连阳天数(30%)
  用 kline_daily 估算

基本面 = 非ST(50%) + 价格合理性(25%) + 流动性(25%)
  用 stock_list + kline_daily 简单判断
```

**这个版本现在就能实现，不需要任何额外数据。**
